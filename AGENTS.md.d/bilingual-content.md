# Bilingual Content Convention

- Use the same file for both languages whenever practical.
- Order content as English first, Russian second.
- Keep headings, bullets, and cross-references parallel across both languages.
- Translate meaning, not just wording; tighten phrasing when a literal translation hurts clarity.
- Keep commands, filenames, URLs, and code samples shared between both language sections.
- When a source artifact exists in only one language, explain its role in the other language nearby instead of duplicating the artifact.
