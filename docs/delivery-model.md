# Delivery Model

## English

### 1. Receive a draft

The client sends a rough spec, backlog dump, or scope note. It only needs to be good enough to start.

### 2. Expand the spec in discussion

Calls, Zoom sessions, screen shares, chats, and comments expose the real needs, constraints, and edge cases.

### 3. Record the working context

Those discussions are recorded and linked to transcripts, notes, comments, and current demo state. That becomes the AI context ledger.

### 4. Generate and deploy

The pipeline generates code, UI, content, automation, tests, and a deployable demo. The client gets a real version, not a promise.

### 5. Review and repeat

Feedback from the live demo becomes the next context layer. The spec keeps moving through evidence.

## Русский

### 1. Получить черновик

Клиент присылает грубое ТЗ, выгрузку backlog или заметку о scope. Этого должно хватить, чтобы начать.

### 2. Раскрыть спецификацию в обсуждении

Звонки, Zoom-сессии, screen share, чаты и комментарии вскрывают реальные потребности, ограничения и edge case'ы.

### 3. Записать рабочий контекст

Эти обсуждения записываются и связываются с транскриптами, заметками, комментариями и текущим состоянием демо. Так появляется AI-реестр контекста.

### 4. Сгенерировать и задеплоить

Конвейер генерирует код, UI, контент, автоматизацию, тесты и deployable demo. Клиент получает реальную версию, а не обещание.

### 5. Проверить и повторить

Фидбек по живому демо становится следующим слоем контекста. Спецификация движется на фактах.
